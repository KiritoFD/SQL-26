# 简易朋友圈平台测试计划

## 1. 测试目标

在实现前先定义验收标准，确保后续代码围绕需求文档开发。测试覆盖数据库设计、用户端功能、管理员端功能、事务、触发器、视图、错误处理和初始化数据。

## 2. 测试分层

### 静态契约测试

静态契约测试不连接 MySQL，只检查项目文件是否包含规定的数据库对象、程序入口和关键实现标记。该类测试用于在开发过程中快速发现遗漏。

对应文件：`tests/test_static_contract.py`

### 数据库集成测试

数据库集成测试需要本地 MySQL 服务。执行建库、建表、初始化数据后，通过 SQL 和命令行逻辑验证外键、触发器、视图和事务效果。

该类测试可在验收前手工执行，也可后续扩展为自动化脚本。

### 命令行交互测试

通过人工输入或模拟输入，验证菜单是否清晰、输入错误是否可恢复、不同角色是否只能访问自己的功能。

## 3. 测试环境

- Python 3.10 或以上。
- MySQL 8.0 或以上。
- `mysql-connector-python`，优先使用仓库 `_vendor` 目录。
- 默认数据库连接参数：
  - `MYSQL_HOST=127.0.0.1`
  - `MYSQL_PORT=3306`
  - `MYSQL_USER=root`
  - `MYSQL_PASSWORD=123456`
  - `MYSQL_DATABASE=moments_lab`

## 5. 数据库对象测试

- `users`、`admins`、`friend_groups`、`friendships`、`moments`、`comments`、`audit_logs` 表必须存在。
- `v_admin_moments` 视图必须存在，并且不能包含普通用户个人基本信息字段。
- `v_friend_moments` 视图必须存在，并且基于好友关系展示朋友圈。
- 至少存在两个触发器：
  - 修改朋友圈内容时刷新 `updated_at`。
  - 插入评论时刷新对应朋友圈 `updated_at`。
- 关键外键必须使用级联删除，支撑删除朋友圈自动删除评论、注销用户清理相关数据。

## 6. 事务测试

### 注册事务

模拟默认分组创建失败时，用户记录不得残留。

### 添加好友事务

模拟第二条双向好友关系插入失败时，第一条关系不得残留。

### 删除好友事务

任一方向删除失败时，另一方向不得被单独删除。

### 管理员注销事务

写审计日志与删除用户必须保持一致。异常时应回滚。

## 7. 错误处理测试

- MySQL 服务不可用时，程序提示检查连接参数。
- 登录失败时，程序不进入角色菜单。
- 输入非数字 id 时，程序提示重新输入。
- 违反唯一约束时，程序提示“已存在”类错误。
- 违反外键约束时，程序提示目标数据不存在或状态不可用。
- 事务失败时，程序回滚并显示错误原因。

## 8. 测试执行顺序

1. 运行静态契约测试：`python -m unittest discover -s tests`
2. 初始化数据库：运行程序中的初始化菜单，或执行 `sql/schema.sql` 与 `sql/seed.sql`。
3. 按功能测试用例进行人工验收。
4. 查看数据库表、视图和审计日志确认结果。
