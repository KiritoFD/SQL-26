# 简易朋友圈平台需求文档

## 1. 背景与目标

本实验要求基于一门高级语言和 MySQL 数据库，搭建一个简易朋友圈平台。系统需要支持普通用户与管理员两类角色，并体现数据库课程中的表结构设计、完整性约束、视图、事务、触发器、增删改查和错误处理。

本项目拟采用 Python 命令行程序作为交互方式，MySQL 作为持久化数据库。

## 2. 角色定义

### 普通用户

普通用户可以维护自己的账号、个人资料、好友关系、朋友圈内容和评论。

普通用户只能查看自己好友的朋友圈，不能查看非好友朋友圈。

### 管理员

管理员可以维护自己的基本信息，审核全部朋友圈，并注销普通用户。

管理员不能浏览普通用户的个人基本信息，审核朋友圈时只能看到朋友圈内容、作者 id、发布时间、最后更新时间和评论数量等必要信息。

## 3. 功能需求拆解

### R-USER-001 用户注册

- 用户输入用户 id、密码、姓名、性别、出生日期、年龄。
- 系统创建用户账号。
- 系统为新用户创建默认好友分组。
- 用户 id 不允许重复。
- 注册过程需要事务保证：用户创建和默认分组创建必须同时成功或同时失败。

### R-USER-002 用户登录

- 用户输入用户 id 和密码。
- 系统校验账号存在、密码正确、账号状态为可用。
- 登录失败时给出明确错误提示。

### R-USER-003 修改个人信息

- 用户可修改姓名、性别、出生日期、年龄。
- 年龄为空或在 0 到 150 之间。
- 性别只能为男、女、其他。

### R-FRIEND-001 搜索用户

- 用户可根据用户 id 或姓名关键字搜索其他可用用户。
- 搜索结果不展示密码。

### R-FRIEND-002 添加好友

- 用户可将其他可用用户添加为好友。
- 用户不能添加自己。
- 已存在好友关系时不能重复添加。
- 添加好友时需要指定本人的好友分组。
- 系统同时建立双向好友关系，事务保证两条关系同时成功或失败。

### R-FRIEND-003 删除好友

- 用户可删除已有好友。
- 删除时系统同时删除双方好友关系。
- 删除过程需要事务保证。

### R-FRIEND-004 好友分组管理

- 用户可创建好友分组。
- 同一用户下分组名不能重复。
- 用户可移动好友到自己的其他分组。

### R-MOMENT-001 发表朋友圈

- 用户可发表朋友圈。
- 朋友圈内容不能为空。
- 朋友圈内容长度不超过 280 字。
- 系统记录创建时间和最后更新时间。

### R-MOMENT-002 修改朋友圈

- 用户只能修改自己的朋友圈。
- 修改后自动刷新最后更新时间。
- 内容限制与发表朋友圈一致。

### R-MOMENT-003 删除朋友圈

- 用户只能删除自己的朋友圈。
- 删除朋友圈时，与该朋友圈相关的评论自动删除。
- 该能力通过外键级联删除体现。

### R-MOMENT-004 查看好友朋友圈

- 用户可查看好友朋友圈列表。
- 列表包含朋友圈 id、好友 id、内容、创建时间、最后更新时间、评论数量。
- 用户可查看某条朋友圈的全部评论。

### R-COMMENT-001 评论朋友圈

- 用户可评论好友朋友圈。
- 评论内容不能为空。
- 评论内容长度不超过 140 字。
- 插入评论后自动刷新对应朋友圈最后更新时间。

### R-ADMIN-001 管理员登录

- 管理员输入管理员 id 和密码。
- 系统校验账号存在且密码正确。

### R-ADMIN-002 修改管理员信息

- 管理员可修改姓名和联系方式。

### R-ADMIN-003 浏览全部朋友圈

- 管理员可浏览全部朋友圈用于审核。
- 管理员不可通过该功能查看用户姓名、性别、出生日期、年龄等个人基本信息。
- 推荐通过审核视图实现字段隔离。

### R-ADMIN-004 删除朋友圈

- 管理员可删除任意朋友圈。
- 删除时相关评论自动删除。
- 系统记录审计日志，包括管理员 id、操作类型、目标朋友圈 id、原因和时间。

### R-ADMIN-005 注销用户

- 管理员可注销或删除普通用户。
- 删除系统中所有与该用户有关的信息，包括好友关系、分组、朋友圈、评论等。
- 系统记录审计日志，包括管理员 id、操作类型、目标用户 id、原因和时间。
- 注销过程需要事务保证：审计日志和用户相关数据清理必须一致。

## 4. 数据库要求映射

| 实验要求 | 对应设计 |
| --- | --- |
| 表结构 | 用户、管理员、好友分组、好友关系、朋友圈、评论、审计日志 |
| 完整性约束 | 主键、外键、唯一约束、CHECK、ENUM、NOT NULL |
| 视图 | 管理员朋友圈审核视图、好友朋友圈查看视图 |
| 事务管理 | 注册、添加好友、删除好友、管理员删除/注销 |
| 触发器 | 修改朋友圈刷新时间、插入评论刷新时间 |
| 增删改查 | 用户、好友、朋友圈、评论、管理员审核 |
| 错误处理 | 登录失败、重复数据、外键失败、长度越界、数据库连接失败 |
| 交互方式 | Python 命令行菜单 |
| 初始化数据 | 至少 1 位用户、1 位管理员，并初始化朋友圈与评论 |

## 5. 非功能要求

- 程序需提供清晰的命令行菜单和输入提示。
- 数据库连接参数应可通过环境变量配置。
- SQL 操作应优先使用参数化查询，避免拼接用户输入。
- 关键数据库操作出现异常时应回滚事务并提示原因。
- 文档需与实现同步维护，便于验收时说明设计依据。

## 6. 待确认事项

- 密码是否需要真实加盐哈希：实验中可使用 `SHA2(password, 256)` 或 Python `hashlib.sha256` 简化实现。
- “注销用户”采用物理删除还是状态禁用：实验原文要求删除系统中所有与用户有关的信息，因此默认采用物理删除，并记录审计日志。
- 好友关系是否需要对方确认：原文只要求添加好友，默认直接建立双向好友关系。
- 管理员是否可评论朋友圈：原文未要求，默认不支持。
