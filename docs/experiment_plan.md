# 简易朋友圈平台实验计划

## 1. 实验目标

基于 Python 高级语言和 MySQL 数据库，实现一个命令行交互的简易朋友圈平台。系统覆盖普通用户端和管理员端，重点体现数据库表结构设计、完整性约束、视图、触发器、事务管理、增删改查逻辑以及错误处理。

## 2. 功能范围

### 用户端

- 注册账号：初始化用户 id、密码和基础资料。
- 登录账号：验证用户 id 与密码。
- 修改个人基本信息：姓名、性别、出生日期、年龄。
- 好友管理：搜索用户、添加好友、删除好友、按分组管理好友。
- 朋友圈管理：发表、修改、删除自己的朋友圈，内容有字数限制，并记录最后更新时间。
- 查看与评论：查看好友朋友圈、评论好友朋友圈，展示最后更新时间和已有评论。

### 管理员端

- 管理员登录：管理员 id 与密码验证。
- 修改管理员基本信息。
- 注销用户：删除系统中所有与该用户有关的信息。
- 朋友圈审核：管理员不可浏览用户个人基本信息，但可以浏览全部朋友圈并删除违规内容。

## 3. 数据库设计计划

数据库名：`moments_lab`

计划建立以下核心表：

- `users`：普通用户账号、密码、姓名、性别、出生日期、年龄、状态。
- `admins`：管理员账号、密码、姓名、联系方式。
- `friend_groups`：用户的好友分组。
- `friendships`：好友关系与所属分组。
- `moments`：朋友圈内容、作者、创建时间、最后更新时间。
- `comments`：朋友圈评论、评论者、评论时间。
- `audit_logs`：管理员删除朋友圈、注销用户等操作日志。

完整性约束计划：

- 主键约束保证实体唯一性。
- 外键约束维护用户、好友、朋友圈、评论之间的引用关系。
- 唯一约束防止重复好友关系和重复分组名。
- `CHECK` 约束限制性别、年龄、朋友圈字数、评论字数等取值范围。
- `ON DELETE CASCADE` 用于朋友圈删除时自动删除评论、用户注销时清理关联数据。

视图计划：

- `v_admin_moments`：管理员审核用视图，只暴露朋友圈信息和作者 id，不暴露用户姓名、性别、生日、年龄等个人资料。
- `v_friend_moments`：好友朋友圈展示视图，包含朋友圈最后更新时间和评论数量。

触发器计划：

- 修改朋友圈内容时自动刷新 `updated_at`。
- 插入评论后自动刷新对应朋友圈的 `updated_at`。
- 删除朋友圈前写入审核/删除日志可由应用层配合事务完成，触发器主要保证时间字段一致性。

事务计划：

- 用户注册：创建用户与默认好友分组放在同一事务中。
- 添加好友：双方好友关系同时插入，任一步失败则回滚。
- 删除好友：双方好友关系同时删除，任一步失败则回滚。
- 管理员注销用户：记录日志并删除用户，失败则回滚。

## 4. 交互实现计划

采用命令行菜单：

- 主菜单：用户注册、用户登录、管理员登录、初始化数据库、退出。
- 用户菜单：资料维护、好友管理、朋友圈管理、查看好友朋友圈、评论。
- 管理员菜单：资料维护、浏览朋友圈、删除朋友圈、注销用户、查看审计日志。

代码使用 `mysql-connector-python`，优先加载仓库内 `_vendor` 目录中的连接器。数据库连接参数通过环境变量配置：

- `MYSQL_HOST`，默认 `127.0.0.1`
- `MYSQL_PORT`，默认 `3306`
- `MYSQL_USER`，默认 `root`
- `MYSQL_PASSWORD`，默认 `123456`
- `MYSQL_DATABASE`，默认 `moments_lab`

## 5. 验收数据计划

初始化脚本将创建至少：

- 普通用户：`1001 / user1001`、`1002 / user1002`、`1003 / user1003`
- 管理员：`9001 / admin9001`
- 好友分组、好友关系、朋友圈、评论、审计日志样例数据。
